# S 级所见即所得文档支持

> 状态：格式总览。各格式必须有独立产品规格和验收标准。

## 当前事实

| 格式                        | 当前能力                                                  | S 级状态                          | 规格                                |
| --------------------------- | --------------------------------------------------------- | --------------------------------- | ----------------------------------- |
| Markdown `.md`, `.markdown` | 常用 Markdown 单界面编辑、资源组、安全保存、选区引用和 Agent 全文操作 | 首轮 S 级与可见资源组实现已完成 | `docs/features/markdown-wysiwyg.md` |
| Word `.docx`                | 标题、段落、列表和表格的只读内容预览                      | 尚未设计 S 级编辑                 | 后续专项                            |
| PowerPoint `.pptx`          | 按幻灯片抽取标题和正文的只读内容预览                      | 尚未设计 S 级编辑                 | 后续专项                            |

只读预览、文本抽取或能够用系统应用打开，都不等于 S 级所见即所得编辑。

## S 级共同口径

每个进入 S 级的格式都必须独立回答：

- 哪些结构可以可视化编辑。
- 保存回原格式时保留什么，允许规范化什么。
- 无法保真的内容如何显示、保护和回退。
- 外部修改、保存失败和恢复草稿如何处理。
- Agent 可以读取和修改到什么粒度。
- 大文件和复杂文档的性能边界。

Markdown 的当前确认口径和开发里程碑见
`docs/features/markdown-wysiwyg.md`。DOCX 和 PPTX 不复用 Markdown 的序列化器，也不能因为共享工作台 Tab 或工具栏而被视为同一种文档会话。
