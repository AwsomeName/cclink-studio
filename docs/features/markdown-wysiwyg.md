# Markdown S 级所见即所得编辑器

> 状态：首轮实现完成，已完成自动化与真实应用集中验收。
> 适用格式：`.md`、`.markdown`。
> 关联文档：`docs/features/document-editor.md`、`docs/features/file-type-support.md`。

## 产品结论

Markdown S 级是：

**面向常用 Markdown 的单一所见即所得文档编辑器，加上安全保存和选区带行号发送给 Agent 会话。**

产品形态参考飞书文档：

- 用户始终在一个可视化文档界面中编辑。
- 不提供 Markdown 源码模式。
- 不提供 Frontmatter、Mermaid 或原始内容专用块。
- 只承诺支持本文明确列出的常用 Markdown。
- 检测到范围外语法时不进入编辑态、不改写草稿、不允许保存覆盖原文。

## 支持范围

首轮支持：

- H1-H6 标题和正文。
- 粗体、斜体、删除线和行内代码。
- 有序列表、无序列表、嵌套列表和任务列表。
- 引用、分隔线和围栏代码块。
- 围栏代码块的语言标识和语法高亮。
- 链接插入、修改和移除。
- 图片插入、替代文本、标题和地址。
- GFM 表格插入、增删行列和单元格编辑。
- 撤销、重做和常用 Markdown 快捷输入。

Mermaid 围栏代码只作为普通代码块显示，不渲染图表。

首轮不支持：

- Frontmatter。
- Mermaid 图表渲染和编辑。
- 原始 HTML。
- MDX/JSX。
- 数学公式。
- 脚注。
- Markdown directives 和其他方言扩展。
- Word 式字体、字号、颜色和分页。
- 图片裁剪、浮动环绕和任意像素定位。
- 高级表格公式、排序和合并单元格。

## 范围外文件

检测到范围外语法时：

1. 保留磁盘文件和工作台原始缓冲区。
2. 不创建替代内容块。
3. 不把范围外内容转换成普通正文。
4. 不进入可编辑状态。
5. 显示具体的不支持原因。
6. 禁止保存覆盖原文件。

范围外语法以后是否支持，按具体格式专项讨论，不预留半成品入口。

## 本地图片

支持文件选择、拖入和剪贴板粘贴。

本地图片默认复制到：

```text
<Markdown 文件名>.assets/
```

规则：

- Markdown 文件和同级可见资源目录共同组成一个“Markdown 文档资源组”。
- Markdown 写入相对路径。
- 文件名冲突时自动改名，不覆盖现有图片。
- 网络图片保持原地址。
- 渲染进程通过安全 IPC 管理本地文件。
- 未命名文档插入本地图片前，先要求保存 Markdown。
- 旧版 `.assets/<Markdown 文件名>/` 在下一次保存、另存为或成组移动时安全迁移；迁移失败不得破坏原引用。

目录示例：

```text
项目/
├── 方案.md
└── 方案.assets/
    ├── manifest.json
    ├── 架构图.png
    └── 截图.jpg
```

Markdown 保持标准相对引用：

```markdown
![架构图](方案.assets/架构图.png)
```

## Markdown 文档资源组

### 磁盘格式

`.md` 是正文唯一事实源，必须能脱离 CCLink 被普通 Markdown 工具读取。资源目录及其中附件均为普通可见文件，不使用点号隐藏目录，不把编辑态正文封装进 `.mdpack`。

存在受管资源时，磁盘文件头可写入一条受控、不可见的依赖声明：

```markdown
<!-- cclink-document: {"version":1,"resources":"方案.assets/manifest.json"} -->
```

- 声明只用于资源完整性提示，不是打开正文的硬依赖。
- 所见即所得编辑器不显示也不编辑该声明。
- 编辑器保存声明占用的源码行偏移，选区发送和点击回跳仍使用磁盘文件的真实行号。
- 仅允许这一种受控注释；普通 HTML 仍属于不支持范围。
- 第三方工具删除声明后，CCLink 仍须通过 Markdown 相对引用检查缺失资源。

`manifest.json` 记录格式版本、所属 Markdown、资源相对路径、媒体类型、大小和 SHA-256。Markdown 正文不复制进 manifest，避免双事实源。

### 资源完整性

打开 Markdown 时检查：

- 声明的资源目录或 manifest 是否缺失、无效。
- Markdown 引用的本地资源是否存在。
- 本地引用是否仍位于同名受管资源目录；目录外引用会提示先收集资源，并阻止 ZIP 产生断链包。
- manifest 中资源的大小或 SHA-256 是否变化。
- 资源目录中是否存在未被正文引用的孤立文件。
- 是否仍在使用旧版隐藏资源目录。

缺失或变化只产生明确诊断，不阻止打开仍然安全的正文。用户可以定位资源目录、重新关联、替换、忽略或清理；第一版不得自动删除孤立资源。

### 成组文件操作

在 CCLink 内执行另存为、复制、移动或重命名时：

- Markdown 与同名 `.assets` 目录作为一个逻辑资源组处理。
- 自动改写受管资源相对路径、依赖声明和 manifest。
- 不覆盖目标位置已有文件或资源目录。
- 操作失败时回滚，原文档和原资源必须仍可使用。
- 打开的编辑 Tab、草稿、Agent 挂载引用和文件树路径同步更新。
- 删除 Markdown 时询问“仅正文”或“正文和资源”，并移到系统废纸篓，不做不可恢复的静默删除。

在 Finder 或其他文件管理器中单独移动 `.md` 无法被 CCLink 阻止；重新打开时必须通过依赖声明和引用扫描提示缺失资源。

### 压缩与导出

日常编辑格式保持 `.md + .assets/` 透明结构。分享和归档时一键导出标准 ZIP：

```text
方案.zip
└── 方案/
    ├── 方案.md
    └── 方案.assets/
        ├── manifest.json
        └── ...
```

- ZIP 是导出物，不是编辑事实源。
- 解压后仍是普通 Markdown 和普通资源文件。
- 导出前检查缺失资源；有缺失时拒绝生成看似完整的压缩包。
- ZIP 条目必须使用相对路径，禁止路径穿越和符号链接。
- PNG、JPEG、WebP、音视频通常已压缩，ZIP 主要解决成组传输，不承诺显著减小体积。
- 图片重采样、格式转换和视频转码属于独立的“优化资源”功能，不在保存时静默执行。

## 保存与恢复

- `Cmd/Ctrl+S` 明确写入磁盘。
- 编辑过程只镜像工作台恢复草稿，不自动覆盖磁盘文件。
- 保存前比较文件版本或内容指纹。
- 外部修改冲突提供重新载入、查看差异、另存为和确认覆盖。
- 写入失败后保留正文和 dirty 状态。
- 磁盘写入采用临时文件和原子替换。
- 打开但未编辑文档不得产生 dirty。
- 解析或序列化结果异常缩减时必须阻止保存。
- 保存包含受管资源的 Markdown 时同步刷新 manifest；manifest 可重建，正文仍是唯一事实源。

## 选区发送到会话

用户选中文档内容后，可通过浮动工具条或右键菜单执行“带行号发送给会话”。

片段包含：

```ts
interface MarkdownFileRangeRef {
  type: 'file-range'
  format: 'markdown'
  path?: string
  tabId: string
  startLine: number
  endLine: number
  startColumn: number
  endColumn: number
  selectedText: string
  sourceSnapshot: string
  snapshotHash: string
  dirty: boolean
}
```

行为：

- 片段进入当前会话输入区，但不自动发送。
- 一条消息可以附加多个片段。
- 消息发送成功后清除待发送片段。
- 已发送消息保留不可变快照。
- 点击片段可以回到文档并定位对应内容。
- 未保存文档使用当前内存内容和 `tabId` 定位。

限制：

- 单片段最多 200 行或 32KB。
- 单条消息的全部片段最多 64KB。
- 限制在界面、协议和主进程三层执行。

## 技术方案

```text
磁盘 Markdown
    ↓
支持范围检测
    ├── 范围外：保持原文，拒绝进入编辑态
    └── 范围内：进入文档解析
                 ↓
         Tiptap / ProseMirror
                 ↓
           所见即所得编辑
                 ↓
        Markdown 序列化与校验
                 ↓
        版本检查和原子保存
```

每个文档会话保存：

- 磁盘原文。
- 当前 Markdown。
- ProseMirror 文档。
- dirty 状态。
- 文件版本指纹。
- 解析和保存诊断。
- 选区到 Markdown 行号的映射。

源码行号映射是内部能力，不产生源码编辑界面。

## 数据安全红线

以下条件全部满足后，改动才能接入真实编辑器：

- 真实长文档打开后正文完整。
- 打开但不编辑不产生 dirty。
- 打开后立即保存不丢失语义内容。
- 范围外语法不进入编辑态。
- 解析异常不修改工作台草稿。
- 空内容或异常缩减内容不能写回磁盘。
- 保存前后必须通过内容规模、块数量和关键结构校验。
