# FPC 外形 AI 改版助手

> 状态：实施中
> 最后更新：2026-07-15
> 关联文档：`docs/features/hardware-workspace.md`、`docs/features/cad-conversion-plugins.md`、`docs/features/product-milestones.md`

## 结论

本需求的目标不是“让 AI 写一份给别人看的改版说明”。用户当前没有其他硬件工程师支持，真正目标是：

```text
CCLink Studio / AI 直接带用户完成 FPC 排线外形调整：
先读取、展示、渲染相关硬件文件；
再按用户自然语言要求修改文件副本；
最后检查新版文件是否还能生产。
```

产品核心能力只有两条：

1. **读得懂、看得见**：读取、展示、渲染 FPC/PCB 相关文件，让用户和 AI 都能看见当前排线、焊盘、连接器、外形、BOM、坐标和 Gerber 层。
2. **改得动、可回退**：按用户要求修改文件，但必须写入副本、新版本目录或变更草案，展示 diff/预览/风险，再由用户确认。

CCLink Studio 在这个场景里的角色是：

```text
AI FPC 改版工程师 + 可视化审查员 + 生产风险检查员
```

## 用户背景

用户有一个 AI 眼镜项目。FPC 排线最初由外部人员制作，但原制作方不提供原理图、EDA 源工程或完整设计资料。

用户后来找人抄板，拿到了生产文件，例如：

- Gerber zip。
- BOM。
- 坐标文件。
- 可能有装配图、PDF、结构图、照片或外壳尺寸。

用户现在想调整 FPC 排线的形状并重新生产。用户不懂硬件，不知道哪些地方能动，哪些地方不能动，但希望 AI 直接带着自己改，而不是再依赖外部硬件工程师。

## 产品目标

CCLink Studio 要把“我想把这个 FPC 形状改一下”变成一个 AI 可执行工作流：

```text
打开 FPC 项目
→ 读取生产文件
→ 渲染外形和关键区域
→ 用户用自然语言/标注说要怎么改
→ AI 判断哪些改动可自动执行，哪些需要补充尺寸或确认风险
→ AI 在副本中修改文件
→ 展示修改前后对比
→ 检查新版生产包
→ 用户确认后再拿去打样
```

第一阶段允许能力不完整，但产品方向必须是“AI 直接读和改”，不是“只生成说明书”。

## 能力分层

### L1：读取与展示

目标：让 AI 和用户知道当前 FPC 文件里有什么。

需要支持：

- 解压 Gerber zip，列出所有层文件。
- 识别机械外形层、铜层、阻焊、丝印、钻孔。
- 读取 BOM 和坐标文件。
- 读取 PDF/图片装配图。
- 读取 DXF/STEP/STP/STL/3MF 等结构约束文件；其中 STEP/STP 需要可选 CAD 转换插件后才能可靠预览。
- 将识别结果组织成项目摘要。

当前已部分实现：

- 已能识别 Gerber zip、BOM、坐标、原理图、结构件等文件。
- 已能解析 csv/tsv/txt/xlsx 的 BOM 和坐标位号。
- 已能生成生产包检查报告。
- 已能在工作区打开 Gerber 层列表，并查看 zip 内单层原始文本预览。
- 已能把常见 Gerber 线段和圆弧解析成几何图元，并在工作区做 SVG 预览。
- 已能从绘制顺序中识别闭合外形候选，计算尺寸、面积、周长和置信度。
- 已能初步分类外轮廓、内孔、开槽、辅助闭合线。
- 已能直接预览 STL/3MF 结构件，STEP/STP 已识别为模型文件但暂不直接渲染。
- Agent 已能通过硬件工具读取指定 Gerber 层的几何线段。

缺口：

- 还不能完整渲染 aperture、region、负片层、阻焊开窗和真实焊盘形状。
- 还不能稳定处理多个外轮廓、嵌套内孔、非绘制顺序连接的轮廓。
- 还不能识别焊盘区域、连接器位置和弯折区域。
- 还没有可选 CAD 转换插件，无法把 STEP/STP 转为可靠预览 mesh，也无法提取结构件包围盒/尺寸供 AI 使用。

### L2：渲染与标注

目标：让用户能看见 FPC，并能指出“这里要改”。

需要支持：

- 渲染 Gerber 机械层/外形层。
- 渲染铜层或至少焊盘/连接器区域。
- 渲染钻孔和关键开窗。
- 渲染结构件参考模型，至少支持 STL/3MF；STEP/STP 通过可选 CAD 转换插件转为预览 mesh。
- 叠加 BOM/坐标位号。
- 允许用户标注：
  - 这里变长。
  - 这里变短。
  - 这里变窄。
  - 这里拐弯。
  - 这里保持不动。
  - 这里是连接器/金手指/焊盘，不能动。
- 保存标注为结构化改版意图。

这一层是 AI 修改文件前的必要基础。用户不懂专业文件，但可以通过“看图 + 标注 + 自然语言”表达目标。

### L3：受控文件修改

目标：AI 按用户要求修改文件。

修改必须分三种路径：

#### 路径 A：源工程修改

如果存在 KiCad、嘉立创 EDA、EasyEDA、Altium 等源工程，优先修改源工程副本。

规则：

- 不覆盖原文件。
- 复制到新版本目录。
- 修改后展示 diff 或变更摘要。
- 重新导出或提示用户导出生产包。

这是最可靠路径。

#### 路径 B：生产文件几何修改

如果没有源工程，只有 Gerber/DXF，允许 AI 做有限的几何级修改，但只在受控范围内：

- 优先修改机械外形层或 DXF 外形文件。
- 尽量不修改铜层走线。
- 如果必须移动/延长铜线，必须进入高风险模式。
- 修改前必须生成可视化预览。
- 修改后必须重新检查 Gerber 包。

适合的低风险改动：

- 外形边缘局部裁切。
- 增加非电气区域延伸。
- 调整不触碰焊盘/走线的机械轮廓。
- 修改 DXF 外壳避让线。

高风险改动：

- 移动连接器、金手指或焊盘。
- 改变线序。
- 让走线变长、变窄或重新绕线。
- 修改弯折区附近铜线。
- 改变阻抗、线宽、间距或层叠。

#### 路径 C：改版草案

如果文件不足以安全修改，AI 仍要生成“可执行改版草案”，但这不是最终产物。

草案应包括：

- 用户意图。
- 当前文件证据。
- AI 判断无法自动修改的原因。
- 建议补充的尺寸、照片或源文件。
- 后续如果补齐文件，AI 可以执行哪一步。

### L4：修改后检查

目标：改完后确认新版生产包是否仍可生产。

检查项：

- Gerber zip 是否完整。
- BOM/坐标位号是否一致。
- 连接器、焊盘、位号数量是否异常变化。
- 外形层是否存在。
- 钻孔文件是否存在。
- 文件版本是否清楚。
- 修改区域是否符合用户标注。
- 是否触碰不可动区域。

第一阶段可以先做文件级和位号级检查；第二阶段做图形级对比；第三阶段做连通性和设计规则检查。

## 用户流程

### 1. 打开 FPC 项目

CCLink Studio 扫描目录，显示：

- 找到了哪些生产文件。
- 是否有源工程。
- 是否有 Gerber、BOM、坐标。
- 是否有外形层线索。
- 当前支持哪些修改路径。

示例：

```text
当前只有 Gerber/BOM/坐标，没有源工程。
可以进入生产文件几何修改模式。
低风险：修改外形轮廓且不碰焊盘/走线。
高风险：移动连接器、改走线、改线序。
```

### 2. 渲染 FPC

用户进入 FPC 预览：

- 左侧文件树。
- 中间 FPC 渲染画布。
- 右侧 AI 改版会话。
- 画布上可以开关层：外形、铜层、钻孔、丝印、坐标点。

如果当前还不能渲染，AI 必须说明原因，并进入“文件清单 + 风险检查”降级模式。

### 3. 用户表达改形状要求

用户可以说：

```text
这段排线太长了，帮我缩短一点。
```

或：

```text
这里需要拐一下，避开外壳里的柱子。
```

或：

```text
这个连接器位置别动，尾巴往右边伸出去。
```

AI 要把需求转成结构化改版意图：

```typescript
interface FpcShapeEditIntent {
  targetRegion: string
  operation: 'extend' | 'shorten' | 'narrow' | 'bend' | 'move-outline' | 'avoid-obstacle'
  fixedRegions: string[]
  dimensions?: {
    lengthDeltaMm?: number
    widthMm?: number
    bendRadiusMm?: number
  }
  riskLevel: 'low' | 'medium' | 'high'
}
```

### 4. AI 执行修改

AI 根据文件条件选择修改路径：

- 有源工程：修改源工程副本。
- 有可识别机械层：修改机械外形层副本。
- 只有不可解释 Gerber：拒绝直接改，要求先渲染/识别或补充文件。

所有修改输出到：

```text
hardware/edits/YYYYMMDD-HHMMSS/
```

目录包括：

- 修改后的文件副本。
- 修改说明。
- 旧版/新版预览图。
- 风险报告。
- 回滚信息。

### 5. 用户确认

CCLink Studio 必须展示：

- 改了哪些文件。
- 改了哪些区域。
- 有没有触碰焊盘、连接器、走线、钻孔。
- 风险等级。
- 新版是否通过生产包检查。

用户确认后，才进入打样准备。

## 产品能力边界

### 可以承诺

- 尽力读取、展示和渲染硬件文件。
- 在能解析的文件格式上执行受控修改。
- 所有修改写入副本，不覆盖原文件。
- 修改前后提供预览、报告和风险提示。
- 对低风险外形修改提供 AI 执行路径。
- 对高风险电气修改明确升级风险并要求确认。

### 不能承诺

- 不能保证无源工程情况下所有 Gerber 修改都安全。
- 不能保证改完后的 FPC 机械寿命、弯折寿命或装配适配。
- 不能保证抄板文件本身正确。
- 不能在无法渲染/无法识别关键区域时盲改。
- 不能绕过工厂 DFM 或生产审核。

### 必须允许 AI 拒绝执行的情况

- 用户要求移动连接器/金手指，但没有线序和连通关系证据。
- 用户要求改走线，但没有源工程或可验证连通性。
- 用户要求缩窄区域，但可能低于 FPC 工艺线宽/间距。
- 用户没有提供机械尺寸，却要求精确避让外壳。
- 文件无法解析，AI 只能猜。

拒绝时不能只说“不行”，必须给下一步：

- 需要补什么文件。
- 需要量哪个尺寸。
- 可以先做哪种低风险修改。
- 当前能否生成草案或标注图。

## 里程碑计划

本路线图的主线不是“下单”，而是：

```text
生产包理解
→ 图形渲染
→ 用户标注/自然语言意图
→ 受控修改副本
→ 修改前后对比
→ 生产风险检查
```

每个里程碑必须能单独验收，不能用“后面 AI 会变聪明”糊过去。

### M0：生产包理解底座

状态：已部分完成。

#### 解决什么问题

AI 不能先改文件，必须先知道项目里有什么文件、哪些文件可能是 FPC 生产文件、哪些文件缺失。

用户不懂硬件，所以系统要先回答：

- 这个目录是不是硬件/FPC 项目？
- 有没有 Gerber、BOM、坐标？
- 有没有源工程？
- 当前能不能进入改形状流程？

#### 怎么解决

主进程新增硬件领域服务：

- 扫描工作空间。
- 识别 Gerber zip、BOM、坐标、原理图、结构件、datasheet。
- 解析 BOM/坐标的 csv、tsv、txt、xlsx。
- 检查 BOM 与坐标位号一致性。
- 生成生产包检查报告。
- 暴露 IPC 和 MCP 工具给 UI 与 Agent。

#### 验收标准

- 打开当前 AI 眼镜/FPC 目录后，能识别硬件生产文件。
- 能指出缺少源工程。
- 能解析 BOM 和坐标位号。
- 能生成 `hardware/reports/*-production-report.md`。
- Agent 能调用硬件工具读取扫描和检查结果。

#### 拷问

如果 AI 连当前有哪些生产文件都说不清，就不能进入“帮我改形状”。

### M1：Gerber 层识别与文件预览

#### 解决什么问题

现在系统只能列出 Gerber zip 和基础层线索，但不能让用户看到 FPC 长什么样，也不能可靠知道哪一层是外形层。

没有层识别，后续修改会变成盲改。

#### 怎么解决

新增 Gerber 层索引：

- 解压 Gerber zip 到受控临时目录。
- 读取 zip 内所有文件名、扩展名和少量内容特征。
- 识别层类型：
  - 外形/机械层。
  - 顶层/底层铜层。
  - 阻焊层。
  - 丝印层。
  - 钻孔层。
- 在 UI 中展示层列表。
- 支持打开单个 Gerber 文本预览和 zip 内容树。

#### 验收标准

- 用户能看到 Gerber zip 内有哪些层文件。
- 系统能给出外形层候选，并标注置信度。
- 系统能给出铜层、钻孔层候选。
- 如果外形层无法识别，必须提示“不能进入自动改外形，只能人工选择层”。

#### 拷问

如果外形层识别错，后续所有渲染和修改都可能错。这个里程碑宁可保守提示人工选择，也不能自信猜错。

### M2：FPC 外形渲染

#### 解决什么问题

用户需要“看见”FPC。没有图形化外形，用户无法表达“这里变短、这里拐弯、这里别动”，AI 也无法验证改动区域。

#### 怎么解决

实现 Gerber 基础渲染：

- 解析常见 Gerber primitives：
  - line。
  - arc。
  - flash/pad。
  - region。
  - aperture。
- 至少渲染外形层、钻孔层和焊盘/铜层轮廓。
- 渲染到 Canvas/SVG。
- 支持缩放、平移、适应窗口。
- 支持层开关。
- 显示坐标单位和边界盒。

第一版可以不追求完整 CAM 级渲染，但必须能渲染当前 FPC 项目的外形。

#### 验收标准

- 用户打开 FPC 项目后能看到外形预览。
- 外形比例和方向基本正确。
- 能开关外形层、铜层、钻孔层。
- 能显示外形尺寸边界。
- 渲染失败时，报告具体不支持的 Gerber 命令或层文件。

#### 拷问

如果只能渲染 demo Gerber，不能渲染用户真实 FPC 文件，这个里程碑不算完成。

### M3：关键区域识别与保护

#### 解决什么问题

AI 可以改形状，但不能乱碰连接器、金手指、焊盘、钻孔和走线。系统必须先标出“高风险不可动区域”。

#### 怎么解决

从 Gerber/BOM/坐标中提取关键区域：

- 从坐标文件提取元件/连接器位置。
- 从铜层/焊盘层识别 pad/flash 密集区域。
- 从钻孔层识别孔位。
- 从丝印或 BOM 中识别 connector/FPC/socket 类器件。
- 在画布上标注：
  - 连接器区域。
  - 焊盘区域。
  - 钻孔区域。
  - 可能弯折风险区域。
- 允许用户手动把区域标记为“不能动”。

#### 验收标准

- 用户能在图上看到至少连接器/焊盘/钻孔候选区域。
- 用户能手动添加“固定区域”。
- AI 在生成修改计划时会引用这些固定区域。
- 涉及固定区域的修改会自动升级为高风险。

#### 拷问

如果不能保护关键区域，就不能放心让 AI 改外形。这个里程碑决定后面修改是否有安全底线。

### M3.5：结构件约束接入

#### 解决什么问题

FPC 形状不是只由 Gerber 决定，还受光机、镜腿、连接件、外壳柱位和装配空间限制。

用户真实需求经常是：

```text
尾巴往右伸，但不要撞到镜腿连接件。
这里缩短一点，要能放进光机左镜框。
连接器位置别动，排线绕开这个壳体台阶。
```

如果 AI 只能看 FPC 自己，无法判断机械避让，就会生成看似可生产但装不进去的改版。

#### 怎么解决

引入结构件约束：

- STL/3MF 直接进入模型预览。
- STEP/STP 通过可选 CAD 转换插件转成 mesh。
- 对结构件提取包围盒、尺寸、三角面数和单位状态。
- 允许用户把结构件加入 FPC 改版任务上下文。
- 用户必须标注或确认 FPC 与结构件的对齐关系；没有装配坐标时 AI 不能假装知道相对位置。
- AI 将结构件约束转为 `MechanicalConstraint`，供后续改版意图引用。

#### 验收标准

- 当前 AI 眼镜目录中的 STL/3MF 能作为结构件上下文打开。
- STEP 在未启用插件时提示需要 CAD 转换；启用 FreeCAD 后可生成预览 mesh。
- AI 能回答“哪些结构件可用于 FPC 避让参考”。
- AI 在缺少装配坐标时会要求用户补充对齐点、尺寸或照片，而不是直接判断干涉。
- `hardware_prepare_fpc_shape_context` 能返回 FPC 外形候选摘要、结构件预览状态、尺寸 metadata、待确认问题和下一步动作。
- 硬件生产侧栏提供“改形状”入口，自动准备上下文包并创建 FPC 形状调整 Agent 会话。
- `FpcShapeEditIntent` 能引用结构件约束和固定区域。

#### 拷问

如果没有坐标对齐，结构件只能作为视觉参考，不能作为可靠干涉结论。这个里程碑必须让 AI 学会承认“不知道装配关系”。

### M4：用户标注与自然语言意图

#### 解决什么问题

用户不懂硬件，不会写“修改机械层坐标”。用户只会说“这里短一点”“尾巴往右伸”“绕开这个柱子”。系统要把这种表达变成结构化修改意图。

#### 怎么解决

新增 FPC 改版交互：

- 用户可在画布上点选、框选、画箭头。
- 用户可用自然语言描述目标。
- AI 结合图上选择和文本生成 `FpcShapeEditIntent`。
- 意图包含：
  - 目标区域。
  - 操作类型。
  - 尺寸参数。
  - 固定区域。
  - 风险等级。
  - 需要补充的问题。

#### 验收标准

- “这里缩短 5mm”能生成结构化 `shorten` 意图。
- “这个连接器别动，尾巴往右伸”能生成固定区域和延伸方向。
- 没有尺寸时，AI 会追问用户如何测量，而不是瞎填。
- 高风险意图不能直接执行，只能进入确认或补充信息流程。

#### 拷问

如果用户仍然必须懂专业术语才能操作，产品就失败了。这个里程碑要把硬件语言翻译成用户语言。

### M5：低风险外形修改副本

#### 解决什么问题

用户的核心目标是“让 AI 去改”。在没有源工程的情况下，第一批可执行修改应从低风险机械外形层开始，而不是直接改走线。

#### 怎么解决

实现受控几何修改：

- 只修改副本，不覆盖原文件。
- 输出到 `hardware/edits/YYYYMMDD-HHMMSS/`。
- 优先修改外形/机械层。
- 支持低风险操作：
  - 外形边缘裁切。
  - 非电气区域延伸。
  - 外形局部平移。
  - 简单避让缺口。
- 修改后重新打包 Gerber zip。
- 自动生成修改报告。

#### 验收标准

- 用户能要求一个低风险外形修改，并得到新版本目录。
- 原始 Gerber zip 不被覆盖。
- 新版外形能重新渲染。
- 修改区域能在图上高亮。
- 如果修改触碰焊盘/走线/钻孔，系统必须阻止或升级为高风险确认。

#### 拷问

如果修改结果只是文本草案，不算完成。这个里程碑必须真的产生文件副本。

### M6：修改前后对比与生产检查

#### 解决什么问题

AI 改完以后，用户需要知道“到底改了什么”“有没有明显改坏”“能不能拿去生产检查”。

#### 怎么解决

新增对比视图：

- 旧版/新版外形叠加。
- 修改区域高亮。
- 层文件变化列表。
- BOM/坐标位号数量对比。
- Gerber 包完整性检查。
- 风险报告写入 `hardware/edits/.../edit-report.md`。

#### 验收标准

- 用户能看到旧版和新版外形差异。
- 用户能看到改动文件列表。
- 用户能看到新版生产包检查结论。
- 用户能确认或放弃本次修改。

#### 拷问

如果用户不知道 AI 改了哪里，就不能信任这个产品。可视化对比是“AI 能改”的信任基础。

### M7：源工程路径与更高风险修改

#### 解决什么问题

低风险外形修改不能覆盖所有需求。用户可能最终需要移动连接器、改变走线或重新布局。这类修改必须回到源工程或更强的设计模型。

#### 怎么解决

按源工程类型逐步支持：

- KiCad：优先支持，因为文本格式开放、可 diff。
- 嘉立创 EDA / EasyEDA：研究工程格式或导入导出路径。
- Altium：早期只读取和提示，不承诺写回。

能力：

- 读取源工程。
- 修改副本。
- 展示结构化 diff。
- 导出或提示导出生产包。
- 对移动连接器、改走线等操作强制高风险确认。

#### 验收标准

- 有 KiCad 源工程时，能在副本里做简单外形修改。
- 修改前后有 diff。
- 修改后能重新生成或提示生成生产包。
- 没有源工程时，不把高风险电气修改伪装成可安全执行。

#### 拷问

如果没有源工程还要移动连接器和改线序，产品必须敢于刹车。AI 不是不能帮，而是要把风险升级到正确层级。

## 当前 CCLink Studio 已支持什么

当前已经支持：

- 扫描硬件项目。
- 识别 Gerber/BOM/坐标/原理图/结构件等文件。
- 检查生产包完整性。
- 解析 csv/tsv/txt/xlsx 的 BOM 和坐标位号。
- 生成生产包检查 Markdown 报告。
- 创建硬件生产检查 Agent 会话。
- Gerber 基础线段/圆弧图形预览、闭合外形候选和尺寸摘要。
- STL/3MF 结构件预览。
- STEP/STP 文件识别和“需要 CAD 转换插件”的明确提示。

当前还不支持：

- 完整 Gerber/FPC 层叠渲染。
- 图上标注修改区域。
- FPC 外形编辑。
- 生成修改文件副本。
- 修改前后图形对比。
- 连通性检查。
- STEP/STP 的 CAD 转换、结构件包围盒提取和装配坐标对齐。

## 拷问

如果产品只能生成说明书，它没有满足用户需求；用户明确没有其他硬件工程师，AI 必须朝“能读、能看、能改文件”发展。

如果产品一上来盲改 Gerber，它同样不合格；正确路径是先渲染和识别，再修改副本，再检查。

如果没有源工程，仍然可以做有限修改，但必须区分低风险机械外形修改和高风险电气修改。

下一步最该实现：FPC/Gerber 外形层渲染和层识别。没有可视化，就没有可靠的 AI 改形状入口。
