# CCLink Studio 工作空间系统

> 当前事实源。最后更新：2026-07-15。

## 结论

CCLink Studio OSS 的工作空间第一阶段只承诺本地工作空间、未归档内容、工作空间内 Tab、草稿、会话和本地文件树。

Remote、CCLink transport、远程文件 provider、远程 Terminal、账号、订阅和配对不属于开源壳默认工作空间模型。旧 Remote 工作空间方案已封存到 `docs/features/historical-remote-workspace-model.md`，不能据此把远程/商业模块加回 Studio OSS。

## 一句话原则

CCLink Studio 以本地工作空间组织工作，以 Tab 承载当前操作，以文件、草稿、会话和诊断记录沉淀结果；没有打开工作空间时，内容进入未归档区域。

## 核心术语

| 术语 | 用户含义 | 当前 OSS 系统含义 |
| --- | --- | --- |
| Home | 继续工作和快速开始 | 最近工作空间、最近 Tab、未归档草稿、未归档会话、待确认任务 |
| 工作空间 | 一件工作事项的本地容器 | 本地文件夹及其工作台状态 |
| 工作空间区 | 左侧管理工作空间的区域 | 最近工作空间、当前工作空间文件树、草稿、会话、未归档入口 |
| 标签页 | 当前打开的工作项 | Browser、Markdown、Android、Terminal、设置、预览、会话 |
| 文件 | 正式保存的成果或资料 | 本地文件系统文件 |
| 草稿 | 尚未保存到具体文件的内容 | 未命名 Markdown 或未保存编辑器状态 |
| 会话 | 用户与 Agent 的协作过程 | 本地 Agent 对话、工具调用记录、任务状态与摘要 |
| 未归档 | 没有明确工作空间归属的区域 | `workspacePath: null` / global workspace |

不新增“项目”“服务器”“远程设备”这类一级对象。用户只需要理解“工作空间、标签页、文件、草稿、会话、未归档”。

## 命名约束

产品文案使用“工作空间”。代码层继续沿用 `workspace`，例如 `workspacePath`、`WorkspaceStateService`、`workspace-state`。

不要为了文案改名全量重命名代码。`workspacePath` 当前表示本地工作空间路径；未来若扩展远程或无文件夹工作空间，需要单独设计兼容层，不能把现有本地状态迁移打断。

## 产品结构

```text
CCLink Studio
├── Home
│   ├── 继续工作
│   ├── 最近工作空间
│   ├── 最近标签页
│   ├── 未归档草稿
│   └── 待确认任务
│
├── 本地工作空间
│   ├── 文件树
│   ├── 草稿
│   ├── 会话
│   ├── Browser Tab
│   ├── Markdown Tab
│   ├── Android / Device Tab
│   └── Terminal Tab
│
└── 未归档
    ├── 未归档会话
    ├── 临时草稿
    ├── 临时下载 / 截图 / 附件
    └── 全局诊断记录
```

## Home

Home 是默认入口，不是普通工作空间，也不是空状态页面。

Home 回答三个问题：

- 我可以从哪里继续？
- 有哪些未处理内容？
- 我可以快速开始什么？

Home 展示：

| 区域 | 内容 |
| --- | --- |
| 继续工作 | 最近工作空间、最近标签页、最近会话 |
| 未处理 | 未保存草稿、未归档会话、Agent 等待确认、失败任务 |
| 快速开始 | 新建 Markdown、打开网页、新建 AI 会话、打开本地文件夹 |
| 诊断 | 最近失败任务、可复制诊断日志入口 |

## 工作空间区

工作空间区不是单纯文件树。它的结构是：

```text
工作空间区
├── 当前工作空间
│   ├── 文件
│   ├── 草稿
│   └── 会话
│
├── 最近工作空间
│   ├── ~/project-a
│   └── ~/cclink-studio
│
└── 未归档
    ├── 未归档会话
    └── 临时草稿
```

用户选择本地工作空间后：

- 左侧展示该工作空间文件树、草稿和会话。
- 中间恢复该工作空间自己的 Tab 集合。
- 右侧 Agent 围绕当前工作空间和当前 Tab 协作。

约束：

- 同一时间只激活一个工作空间。
- 文件树只展示当前激活工作空间。
- Tab 列表由主工作区顶部 Tab Bar 管理，不塞进侧栏。
- 未归档固定在工作空间区底部，不伪装成普通文件夹。
- 设置、诊断这类全局页面不进入工作空间状态快照。

## 工作空间状态

当前代码已经具备工作空间状态基础：

| 当前实现 | 产品含义 |
| --- | --- |
| `workspacePath` | 当前本地工作空间路径 |
| `workspacePath: null` | 未归档 / global workspace |
| `WorkspaceState` | 工作台状态快照 |
| `tabs` | 当前工作空间内标签页 |
| `browserTabs` | 当前工作空间内浏览器状态 |
| `editorDrafts` | 当前工作空间内草稿和未保存文件 |
| `agentConversations` | 当前工作空间内会话 |
| `fileTree` | 当前工作空间内文件树状态 |

切换工作空间时必须：

1. 保存当前工作空间状态快照。
2. 隐藏当前工作空间 Tab。
3. 加载目标工作空间状态快照。
4. 目标工作空间没有快照时，使用干净工作台，不继承上一个工作空间现场。
5. 保留全局 Tab，例如设置或诊断。

## 草稿模型

草稿解决的是“先写，稍后决定保存位置”。

归属规则：

- 有当前工作空间时，新草稿归属当前工作空间。
- 没有当前工作空间时，新草稿归属未归档。
- 切换工作空间不会把草稿带走。
- 保存到文件后，草稿变成本地工作空间文件。

关闭规则：

- 空白未命名草稿：关闭时静默丢弃。
- 有内容未命名草稿：关闭时提供“保存到文件 / 保留草稿 / 丢弃”。
- 已关联文件且有修改：关闭时提供“保存 / 不保存 / 取消”。

第一阶段只需要保证草稿不因工作空间切换丢失；移动、批量整理、长期归档可后置。

## 会话模型

会话是可恢复的本地工作过程，不只是右侧聊天消息。

归属规则：

- 有当前工作空间时，新会话归属当前工作空间。
- 没有当前工作空间时，新会话归属未归档。
- 会话可以从右侧 Agent Panel 发起，也可以作为 Workbench Tab 恢复。
- 会话历史列表属于工作空间区或 Home，不塞满右侧 Agent Panel。

会话最少需要记录：

- `id`
- `title`
- `workspacePath | null`
- `createdAt`
- `updatedAt`
- `archivedAt?`
- 消息摘要或消息引用
- 最近工具调用状态

## 标签页边界

工作空间标签页：

- Markdown
- Browser
- Android / Device
- Terminal
- Preview
- Conversation
- Draft

全局标签页：

- Home
- Settings
- Diagnostics
- About / Help

全局标签页不随工作空间切换关闭。工作空间标签页必须随工作空间状态保存和恢复。

## 本地 Terminal

Terminal 是当前工作空间里的高风险 Tab 类型。

约束：

- 本地工作空间打开本机 shell。
- Terminal 默认继承当前工作空间路径。
- 高风险命令进入权限确认和审计链路。
- 关闭 Terminal Tab 时必须定义进程生命周期：终止、保持后台或转为只读记录。
- OSS 默认不承诺远程 Terminal；远程执行由 commercial/CCLink runtime 另行注入。

## 分阶段实现

### Phase 1：术语和入口

- UI 文案从“文件区”收敛为“工作空间区”。
- 工作空间第一版等同本地文件夹，继续复用 `workspacePath`。
- 工作空间区展示当前工作空间路径、文件树、草稿、会话。
- 保持内部 Activity ID 稳定，避免无意义迁移。

### Phase 2：状态隔离

- 维护最近工作空间列表。
- 切换工作空间时保存/恢复 Tab、browserTabs、editorDrafts、agentConversations、fileTree。
- 目标工作空间没有状态时使用空工作台。
- 全局 Tab 不进入工作空间快照。

### Phase 3：未归档

- 复用 `workspacePath: null` 作为未归档区域。
- 没有当前工作空间时，新草稿和新会话进入未归档。
- Home 展示未归档草稿和会话。
- 未归档固定在工作空间区底部。

### Phase 4：草稿恢复

- 工作空间区草稿列表可点击恢复。
- 已打开草稿激活对应 Tab。
- 已关闭草稿重新打开为未命名 Markdown Tab。
- 支持保存到当前工作空间文件。

### Phase 5：会话治理

- 工作空间区展示当前工作空间会话。
- Home 展示最近会话和未归档会话。
- 支持归档、恢复、删除本地会话视图。
- 会话与当前工作空间路径稳定绑定。

## 不做什么

当前 OSS 工作空间模型不做：

- 远程工作空间。
- CCLink transport。
- 远程文件树。
- 远程 Terminal。
- 远程 Agent 面板。
- 账号、订阅、entitlement、配对。
- 云同步工作空间状态。
- 把 `workspacePath` 机械替换为新对象模型。

这些属于历史方案或 commercial/CCLink overlay，不应回流到 Studio OSS 默认路径。

## 拷问

第一问：本地工作空间等同文件夹是不是太窄？是，但它是当前 OSS 最稳的可交付边界。未归档先兜住无文件夹场景，远程/云端后置。

第二问：为什么不继续保留 Remote 模型？因为相关代码和商业边界已经迁出。文档继续把 Remote 写成当前计划，会诱导把 `remote`、`cclink`、`entitlement` 文件加回开源壳。

第三问：为什么不重命名 `workspacePath`？因为这是现有状态恢复和兼容路径。先稳住产品文案，再做兼容迁移设计。

第四问：最大的失败路径是什么？切换工作空间时丢草稿、丢会话、继承上一个工作空间现场。这比 UI 名字没改干净更伤用户信任。
