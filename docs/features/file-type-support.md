# 文件类型支持分档与开发计划

本文记录 CCLink Studio 开源桌面壳的文件打开策略。核心原则：能编辑的进入编辑器，能预览的进入只读预览器，不能安全解析的必须明确降级，不能把二进制内容塞进 Markdown 编辑器。

## S 级所见即所得编辑

S 级表示：可以可视化编辑、保存回原格式，并在约定范围内保留主要结构。只读预览不等于 S 级。

当前规划：

- Markdown：`.md`, `.markdown`。产品设计和里程碑已确认，见
  `docs/features/markdown-wysiwyg.md`；常用 Markdown 的首轮 S 级实现与集中验收已完成。正文保持标准独立文件，本地资源使用同级可见的 `<文件名>.assets/`，分享和归档导出标准 ZIP。
- Word：`.docx`。当前只有只读内容预览，S 级编辑后续单独设计。
- PowerPoint：`.pptx`。当前只有只读内容预览，S 级编辑后续单独设计。

## 一级支持：创作素材与日常文件

一级支持表示：双击必须进入合适的打开路径，不能乱码；能被内嵌浏览器解码的文件应直接预览；不能内嵌解码时给系统打开或转码提示；所有文件都应可挂到会话资源。

| 类型            | 扩展名                                                                                                                                                       | 当前策略                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------- |
| 文本/代码       | `.md`, `.markdown`, `.txt`, `.json`, `.jsonl`, `.yaml`, `.yml`, `.toml`, `.xml`, `.html`, `.css`, `.js`, `.jsx`, `.ts`, `.tsx`, `.py`, `.sh`, `.sql`, `.log` | 文本编辑器                                                           |
| 表格文本        | `.csv`, `.tsv`                                                                                                                                               | 文本编辑器                                                           |
| 图片素材        | `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.svg`, `.bmp`, `.ico`                                                                                             | 图片预览                                                             |
| 投稿视频素材    | `.mp4`, `.mov`, `.webm`, `.m4v`                                                                                                                              | 300MB 以下内嵌播放；超过 300MB 明确提示改用系统播放器或后续流式预览  |
| 音频素材        | `.mp3`, `.wav`, `.m4a`, `.aac`, `.flac`, `.ogg`, `.opus`                                                                                                     | 原生可解码格式内嵌播放                                               |
| PDF             | `.pdf`                                                                                                                                                       | 内嵌 PDF 预览                                                        |
| Word 文档       | `.docx`                                                                                                                                                      | 只读内容预览：标题、段落、列表、表格；S 级所见即所得编辑后续单独设计 |
| PowerPoint 文稿 | `.pptx`                                                                                                                                                      | 只读内容预览：按幻灯片抽取标题和文本；S 级所见即所得编辑后续单独设计 |

## 二级支持：专业文件与容器

二级支持表示：应识别文件类型，提供专业预览或目录/摘要；暂不能解析时也要有明确提示和系统打开入口。

| 类型        | 扩展名                                                                         | 当前策略                                                                 |
| ----------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------ |
| 3D/模型     | `.glb`, `.gltf`, `.fbx`, `.stl`, `.3mf`, `.step`, `.stp`, `.obj`, `.mtl`       | 模型预览 Tab                                                             |
| 硬件/Gerber | `.gbr`, `.gtl`, `.gbl`, `.gto`, `.gbo`, `.gts`, `.gbs`, `.gm1`, `.gko`, `.drl` | Gerber/硬件预览                                                          |
| 压缩包      | `.zip`, `.tar`, `.gz`, `.tgz`, `.7z`, `.rar`                                   | 不做内置预览；zip 支持文件树右键解压到同级同名文件夹，其它格式识别后降级 |

## 三级支持：明确降级

三级支持表示：不做内置解析，不进入文本编辑器；只提供系统打开、格式转换建议和会话挂载。

- Office/OpenDocument：`.doc`, `.xls`, `.xlsx`, `.ppt`, `.odt`, `.ods`, `.odp`
- Apple iWork：`.pages`, `.numbers`, `.key`
- 安装包/应用包：`.dmg`, `.exe`, `.pkg`, `.app`
- 冷门视频格式：`.mkv`, `.flv`, `.avi`, `.wmv`, `.mpeg`, `.mpeg4`, `.mpg`, `.ts`, `.rm`, `.rmvb`

## 本轮开发计划

1. 完成一级文件类型分流：图片、PDF、主流视频、音频进入 `file-preview`，文本继续进入编辑器。
2. 完成二级文件类型分流：3D 模型和 Gerber 保持专业预览，压缩包进入明确降级路径。
3. 完成安全降级：Office/OpenDocument、iWork、冷门视频格式不再乱码，显示系统打开、格式转换或后续规划提示。
4. 补齐 zip 操作：zip 不做预览，但文件树右键提供“解压到同名文件夹”，遇到重名目录自动编号，并拦截 zip slip 路径穿越。
5. 补齐测试：文件路由、主进程渲染结果、媒体 300MB 边界、zip 解压成功/重名/非法路径。
6. Markdown S 级按 `docs/features/markdown-wysiwyg.md` 单独推进；DOCX/PPTX 只读预览不能计为 S 级编辑完成。
