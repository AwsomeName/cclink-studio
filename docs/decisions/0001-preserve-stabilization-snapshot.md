# ADR 0001：保留稳定化现场快照提交

- 状态：accepted
- 日期：2026-07-20
- 负责人：CCLink Studio Maintainers

## 问题

提交 `49da3b2` 将稳定化开始前工作树中的 104 个跨域文件作为一个现场快照保存在共享 `main` 历史。它不符合“每个后续工作包可独立审计和回滚”的目标，但改写已经共享的历史会破坏取证基线、提交引用和其他工作树，并可能覆盖用户并行改动。

## 决策

保留 `49da3b2`，不重写 `main` 历史。该提交只被视为稳定化现场快照，不被视为以后跨域提交的范例。

快照内文件的领域归属、权限面、状态所有者、生命周期和验收边界由 `docs/ops/stabilization-s0-inventory.md` 记录。快照之后的修复必须继续按单一目标形成独立提交；需要跨域集成时，必须在提交或 PR 中列明集成点和回滚方式。

## 不变量

- 本 ADR 只覆盖 `49da3b2`，不能用于豁免任何新提交。
- 不以拆分提交为名执行 force-push、重写共享 `main` 或丢弃用户工作树。
- 快照后的每个稳定化提交必须目标明确、可审计，并通过受影响测试。
- 候选基线仍必须满足干净 worktree、完整门禁、Electron smoke 和人工验收要求。
- 若快照中的某个领域需要回滚，必须以有测试保护的领域修复或显式 revert 处理，不能整体回退其他已验证能力。

## 备选方案

1. 重写 `main`，把 `49da3b2` 拆成多个提交。拒绝：破坏共享历史，风险高于审计收益。
2. 忽略跨域问题。拒绝：会让后续开发继续复制不可回滚的提交方式。
3. 整体回退 `49da3b2` 后重新实现。拒绝：会丢失已经通过 718 项测试和 smoke 验证的现场。

## 风险与影响

- `49da3b2` 无法按领域直接 revert；库存文档和后续领域提交是主要审计依据。
- blame 粒度在该快照内较粗，需要结合库存和后续提交理解。
- 接受例外可能被误解为降低标准，因此必须保持“只覆盖该提交”的明确边界。

## 迁移计划

1. 保留 `49da3b2` 和原始库存记录。
2. 所有 S0 收口修复按单一目标提交。
3. 在稳定化退出记录中引用本 ADR。
4. 后续修改快照内代码时，逐步把领域契约、测试和所有权沉淀到正常提交历史。

## 回收或复审条件

- 仓库尚未共享且维护者明确批准历史重写时，可以重新评估；当前条件不成立。
- 如果发现快照包含无法隔离的数据泄露或安全材料，安全处置优先，本 ADR 立即复审。

## 验证

- `git branch --contains 49da3b2` 可以确认快照仍在候选历史中。
- `docs/ops/stabilization-s0-inventory.md` 覆盖快照中的全部领域和跨域文件。
- `49da3b2` 后的 S0 修复按独立目标提交，并通过 `pnpm verify` 与 Electron smoke。
